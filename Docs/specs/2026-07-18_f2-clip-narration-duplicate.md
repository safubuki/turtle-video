# F2: 動画クリップ / ナレーションの簡単コピー - 仕様書

## 概要

追加済みの動画クリップとナレーションを 1 タップで複製できるようにする。
同じ動画の別シーンをカットとして並べる/長いナレーションを分割配置する作業を、
「1 回だけ読み込み → コピー → トリミング」の流れで完結させる。

## 背景・課題

- 同じ動画から複数シーンを使う場合、現在は毎回ファイル選択→読み込みが必要で非常に手間。
- 長いナレーションを複数の位置に分けて使う場合も同様。
- コピー + トリミングで「1 ソース複数カット」を実現したい。

## 要件一覧

| # | 要件 | 優先度 | 説明 |
|---|------|--------|------|
| R1 | 動画/画像クリップの複製 | 必須 | クリップのコピーを直後に挿入。トリム/変形/音量/フェード設定も複製 |
| R2 | ナレーションの複製 | 必須 | クリップのコピーを追加。トリム/音量設定も複製 |
| R3 | 独立性 | 必須 | 複製は独立した ObjectURL を持ち、片方の削除がもう片方に影響しない |
| R4 | 配置ルール | 必須 | 動画: 元の直後に挿入。ナレーション: 元のトリム後末尾に続けて配置（重なり回避） |
| R5 | Android/PC 限定 | 必須 | コピー UI は standard フレーバーのみ表示 |
| R6 | ロック尊重 | 必須 | セクションロック中はコピー不可 |

## 動作仕様

### 動画/画像クリップ（mediaStore.duplicateMediaItem）

- 元アイテムの全プロパティをコピーし、新 id + `URL.createObjectURL(元file)` の新 URL を付与。
- `isTransformOpen: false` / `isLocked: false` で初期化（開閉/ロック状態は複製しない）。
- 元アイテムの直後（index+1）へ挿入し、totalDuration を再計算。
- 保存時は同一ファイルが複製分だけ保存される（重複排除は将来課題として許容）。

### ナレーション（audioStore.duplicateNarration）

- 元クリップの全プロパティをコピーし、新 id を付与。
- `file` が `File` の場合は新 ObjectURL を発行。File でない場合（理論上の縮退データ）はコピー不可。
- `startTime = 元の startTime + 元のトリム後実効長`（直後に連続配置、重なりを避ける）。
- AI ナレーションも同様に複製可能（aiScript 等も引き継ぐ）。

## 画面仕様

- クリップ一覧の各アイテム操作ボタン列に「コピー」アイコンボタン（lucide `Copy`）を追加。
- ナレーション一覧の各クリップ操作ボタン列にも同様に追加。
- 表示条件: standard フレーバー（`!isIosSafari`）かつ非ロック時。
