# F3: 複数 BGM 対応 - 仕様書

## 概要

BGM を複数ファイル追加できるようにし、ナレーションと同様のトリミング・配置操作を提供する。
追加時は動画全体の長さに自動でフィットさせ、手動調整の手間を減らす。

## 背景・課題

- 現在 BGM は 1 ファイル（`bgm: AudioTrack | null`）のみ。
- シーンごとに曲を切り替える動画が作れない。
- ナレーションは既に複数クリップ（`NarrationClip[]`）＋トリムの仕組みを持っており、
  再生・書き出しのパイプラインも汎用化されている。

## 設計方針: 「BGM クリップ = ナレーション型クリップ」として再利用

standard フレーバーの preview エンジン / export エンジンは `NarrationClip[]` を汎用の
「タイムライン配置オーディオクリップ」として処理している。BGM クリップも同じ形にし、
**再生・書き出しへはナレーション配列へマージして流す**（エンジン改修を最小化）。

```ts
// src/types/index.ts
export interface BgmClip extends NarrationClip 相当 {
  // NarrationClip と同形（sourceType: 'file' 固定運用）
  // + フェード（BGM の既存機能を維持）
}
// 実装は NarrationClip に optional フェードフィールドを追加し、BgmClip = NarrationClip の別名とする
export interface NarrationClip {
  // ...既存...
  fadeIn?: boolean;          // 追加（省略時 false）
  fadeOut?: boolean;         // 追加
  fadeInDuration?: number;   // 追加
  fadeOutDuration?: number;  // 追加
}
export type BgmClip = NarrationClip;
```

- `audioStore` に `bgmClips: BgmClip[]` を追加。既存 `bgm`（単一）は後方互換のため型として残す。
- TurtleVideo（standard 時のみ）で `pipelineNarrations = [...narrations, ...bgmClips]` を構成し、
  MediaResourceLoader / preview エンジン / export の音声ソース / preview cache key に渡す。
  ID は `bgmclip_` プレフィックスで衝突回避。
- フェードは standard の preview エンジン（クリップ音量計算）と standard export エンジン
  （OfflineAudioContext エンベロープ）にのみ実装。クリップ範囲基準
  （フェードイン=クリップ先頭から、フェードアウト=クリップ末尾まで）。

## 自動フィット仕様（追加時）

- `startTime =` 既存 BGM クリップの実効末尾（`max(startTime + トリム後長)`）。無ければ 0。
- `trimEnd =` `trimStart + min(ソース長, totalDuration - startTime)` に自動調整
  （動画の残り時間にぴったり収める）。
- `totalDuration === 0`（メディア未追加）または残り時間 ≤ 0 の場合はトリムせず追加
  （後から動画を足すケースを妨げない）。
- 1 本目: 動画全体にフィット。2 本目: 1 本目の終わりから動画末尾までにフィット。

## 旧データ互換 / iOS 互換（ミラー仕様）

- **読み込み時（standard のみ）**: `bgm` があり `bgmClips` が空なら、`bgm` を 1 本の BgmClip へ
  自動変換（startTime=delay, trimStart=startPoint, フェード/音量引き継ぎ）し、以後はクリップとして扱う。
- **保存時（共有）**: `bgmClips` が 1 本以上あれば、先頭クリップを `AudioTrack` 形式へ近似変換して
  `bgm` フィールドにもミラー保存する。→ iOS / 旧バージョンで開いた場合は「1 本目の BGM だけ再生」
  という縮退動作になり、クラッシュや無音を避ける。
- iOS 側 UI / エンジンは一切変更しない（`bgmClips` は無視される）。

## 画面仕様（BgmSection / standard のみ）

- 現在の「1 ファイル選択」UI をクリップリスト形式に変更（apple-safari は現行 UI を維持）。
- 各クリップ: ファイル名 / 開始位置（タイムライン）/ トリム（start・end）/ 音量 / ミュート /
  フェードイン・アウト / 削除 / 上下並び替え — ナレーションクリップと同等の操作感。
- 「＋ BGM を追加」で複数選択可。追加時に自動フィットを適用しトースト等で通知不要（見て分かる）。

## 影響を受けるファイル

| ファイル | 変更内容 |

| `src/types/index.ts` | NarrationClip へ optional フェード追加、`BgmClip` エイリアス |
| `src/stores/audioStore.ts` | `bgmClips` 状態 + add(自動フィット)/update/trim/move/remove/複製 + restore |

| `src/flavors/standard/export/exportEngine.ts` | scheduleNarrationClip へフェードエンベロープ対応 |

| `src/hooks/useAutoSave.ts` | 差分ハッシュへ bgmClips を追加 |

## テスト計画

- audioStore: 自動フィット（1本目/2本目/動画なし/残りゼロ）、復元、ミラー変換の往復。
- export: フェードエンベロープの resolver 純ロジックテスト（可能な範囲）。
- 既存 preview/export テストの回帰確認。

## 制約・将来課題

- BGM クリップの重なり配置は許可（音はミックスされる）。自動フィットは重なりを作らない。
- クロスフェード（曲間の重ねフェード）は将来課題。
- iOS への展開時は apple-safari エンジンのクリップフェード対応 + BgmSection UI 切替を行う。
