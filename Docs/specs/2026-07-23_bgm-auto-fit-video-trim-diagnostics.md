# BGM動画尺自動追従・動画トリム位置合わせ・診断ログ 仕様書

## 概要

BGMの元の配置・トリム設定を保持したまま、現在の動画尺に対する実際の再生区間を自動調整する。
あわせて、動画カードでもプレビュー現在位置を使ったトリム開始・終了設定を可能にし、問題調査用の詳細ログを既存ログモードへ統合する。

## 背景・課題

- 動画を短くした後にBGMの「動画末尾合わせ」を忘れると、フェードアウトが動画末尾までに完了しない。
- 動画尺の外へ出た後続BGMを削除すると、動画を再び長くした際に元へ戻せない。
- 動画のトリム値を数値だけで合わせるのは、映像を確認しながら作業する用途では手間がかかる。
- 最近追加した機能の調査ログを増やしたい一方、標準モードで高頻度ログを記録すると再生性能へ影響し得る。

## 要件一覧

| # | 要件 | 優先度 | 説明 |
|---|------|--------|------|
| R1 | BGM実効区間の自動算出 | 必須 | 保存された開始・トリム値は変更せず、動画末尾で実効終了だけをクランプする |
| R2 | BGMの自動休止・復帰 | 必須 | 開始位置が動画尺外のBGMは再生対象外とするが削除せず、動画延長時に自動復帰する |
| R3 | BGMフェードの末尾追従 | 必須 | 動画末尾で短縮された実効クリップを再生・書き出しへ渡し、フェードアウトを実効末尾基準で適用する |
| R4 | BGM状態の可視化 | 推奨 | 各カードに実効再生区間、自動短縮、自動休止の状態を表示する |
| R5 | 動画トリムの現在位置反映 | 必須 | 対象カードの表示区間内にあるプレビュー位置を、動画ソース内時刻へ変換して開始・終了へ設定する |
| R6 | 動画トリム後の再計算 | 必須 | 操作直後に総尺・各カード範囲を再計算し、以後の位置合わせは短縮後タイムラインを基準にする |
| R7 | 標準ログの低負荷維持 | 必須 | 標準モードでは新しい詳細診断ログを保存せず、警告・エラー等の通常ログ契約を維持する |
| R8 | 詳細診断ログ | 必須 | 詳細モードでBGM実効区間変更とプレビュー動画トリムの変換前後を記録する |

## 動作規約と境界条件

### BGM

- 元設定が `BGM1: 0〜8秒`、`BGM2: 8〜15秒` のとき、動画尺6秒ではBGM1を `0〜6秒` として再生し、BGM2を自動休止する。
- 同じ動画を12秒へ戻すと、BGM1は元の `0〜8秒`、BGM2は `8〜12秒` として自動復帰する。
- 動画尺による自動調整では `startTime`、`trimStart`、`trimEnd` を書き換えない。ユーザーが開始・終了・トリムを直接編集した場合だけ元設定を変更する。
- BGM間の隙間・重なりはユーザーの元設定を尊重し、自動整列や詰め直しは行わない。
- 動画がない、クリップ長が最小再生時間未満、開始位置が動画末尾以降の場合は再生パイプラインから除外する。
- フェードアウトが有効な場合、自動短縮後の実効末尾をフェード終端とする。無効な場合にフェードを勝手に有効化しない。

### 動画トリム

- プレビュー位置が対象カードの現在の表示区間内にある場合だけ操作を有効にする。
- ソース時刻は `現在のtrimStart + (プレビュー位置 - カードのタイムライン開始)` で求め、既存の最小クリップ長規約へクランプする。
- 開始を設定した後は、選択したフレームが新しいカード先頭になるため、プレビュー位置をそのカードのタイムライン開始へ移す。
- 終了を設定した後は、選択位置を新しいカード終端として維持する。ただし新しい総尺を超える場合は既存の総尺クランプに従う。
- ディゾルブ区間ではカードごとのタイムライン開始を使う。重なり中でも、操作したカード自身の範囲を基準に変換する。

### ログ

- 既存の `標準 / 境界診断 / 詳細` の選択を共通の記録量設定として利用する。
- 新規の操作診断は `DEBUG` とし、`詳細` のときだけ保存する。
- `標準` と `境界診断` では、新規の一般DEBUGログを早期returnして、ログ配列更新とsessionStorage書き込みを行わない。
- `境界診断` の既存プレビュー境界INFOログは従来どおり維持する。

## 画面仕様

- BGMカードの再生区間は保存値ではなく、現在の動画尺に対する実効値を表示する。
- 動画末尾で切られているカードには `動画尺に自動調整`、全体が動画尺外のカードには `自動休止` を表示する。
- 破壊的な手動トリムと誤認しないよう、状態表示は操作ボタンではなくバッジと補足文で示す。
- 動画カードのトリミング欄に `プレビュー位置を反映: 開始 / 終了` を置き、対象範囲外では無効理由をtitleで説明する。

## データ構造

保存形式は変更しない。BGMの自動調整は次の派生情報としてメモリ上でのみ扱う。

```ts
interface EffectiveBgmClipState {
  source: BgmClip;
  playbackClip: BgmClip | null;
  effectiveStart: number;
  effectiveEnd: number;
  isInactive: boolean;
  isTrimmedByTimeline: boolean;
}
```

## 実装計画

### Phase 0: 契約固定

**目標**: 純粋関数でBGM実効区間と動画ソース時刻変換の境界条件を固定する。

タスク:
├── BGM実効区間ユーティリティとテストを追加
├── 動画プレビュー位置変換ユーティリティとテストを追加
└── 既存保存形式が変わらないことを確認

### Phase 1: BGM再生・表示統合

**目標**: プレビューと書き出しが同じ実効BGMを使用し、UIが自動調整状態を示す。

タスク:
├── `pipelineNarrations` へ実効BGMのみを統合
├── BGMカードの区間・状態表示を更新
└── 短縮・休止・復帰・フェード終端をテスト

### Phase 2: 動画トリムUI

**目標**: 各動画カードから現在位置を安全に開始・終了へ反映できる。

タスク:
├── currentTimeとコールバックをカードまで伝搬
├── 範囲内判定、ソース時刻変換、開始後の再配置を実装
└── UIとハンドラのテストを追加

### Phase 3: 診断ログ

**目標**: 標準モードの負荷を増やさず、詳細モードで再現調査に必要な値を残す。

タスク:
├── DEBUG記録を詳細モードに限定
├── BGM実効計画と動画トリム変換のログを追加
└── モード別ログテストを追加

### Phase 4: 品質確認・資料更新

**目標**: プレビュー・書き出し・保存を含む回帰を確認し、実装上の注意点を残す。

完了条件:
- 対象テストと全テストが成功する
- TypeScript、ビルド、Lint、品質ゲートが成功する
- Turtle Video overviewへ実装契約を追記する
