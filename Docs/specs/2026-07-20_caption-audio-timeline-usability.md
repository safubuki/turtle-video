# キャプション・音声タイムライン操作改善 仕様書

## 概要

キャプションのまとめて入力と個別設定、BGM／ナレーションの時間調整を、
スマートフォンでも理解しやすく、操作結果が画面表示と一致する形へ改善する。
既存プロジェクトとの互換性を維持し、新しい永続化フィールドは追加しない。

## 背景と課題

- 「空行で区切る（時分割）」から「1行 = 1カード」へ戻しても、入力欄に時分割構造が残る。
- 時分割カードの内部改行を表す `⏎` は、入力方法や意味が分かりにくい。
- 通常カードと時分割カードを同じ一括入力内で簡単に混在させにくい。
- 一括編集の時間記法だけを消して文章を再利用する操作がない。
- 個別設定を既定値へ戻しても、一部の override が残ると「個別設定」表示が消えない。
- BGM／ナレーションは現在のプレビュー位置を開始へ反映できるが、終了へ反映できない。
- BGM を複数配置した後、動画末尾を超えたクリップを手作業でトリムする必要がある。

## 機能要件

### R1: まとめて入力の3モード

1. `1行カード`: 空行以外の各行を別カードとして扱う。
2. `混在（おすすめ）`: 通常行は別カード、行頭が `+ ` の行だけ直前カードの時分割行にする。
3. `空行で時分割`: 空行で区切った各まとまりを1カードにし、まとまり内の複数行を時分割する。

モード切替時は入力欄のテキストも変換する。

- 時分割／混在から `1行カード` へ戻すと、内部の各表示行を独立した1行へ展開する。
- `1行カード` から `空行で時分割` へ移ると、既存カード間へ空行を入れ、意図せず全行が1カードになることを防ぐ。
- 混在と空行モードの相互切替では、カード構造を維持する。
- 既存の `⏎` 入力・保存済みテキストは引き続き解釈できる。

### R2: スマートフォン向け時分割入力

- 混在モードに `＋ 時分割行を追加` ボタンを設け、カーソル位置へ `+ ` 行を挿入する。
- UI説明とプレビューでは `⏎` を入力記号として案内しない。
- プレビューは時分割行を視覚的な矢印で連結して表示する。

### R3: 時間指定だけをクリア

- まとめて入力欄に `時間指定だけ消す` ボタンを追加する。
- 有効な `[開始-終了]` の前置・後置だけを除去し、文章、改行、時分割構造は維持する。
- 時間形式に一致しない本文中の角括弧は変更しない。

### R4: キャプション個別設定のクリア

- 個別設定モーダルに `この個別設定をクリア` ボタンを追加する。
- 対象カードの本文・開始・終了時間・時分割本文は変更しない。
- 対象カードのスタイル override、フェード override、時分割固有設定をすべて `undefined` に戻す。
- クリア直後に対象カードの「個別設定」バッジが消える。
- 個別設定の有無判定とクリア対象を同じユーティリティで管理する。

### R5: プレビュー位置による音声区間調整

- BGM とナレーションの各クリップで、現在のプレビュー位置を開始位置へ設定できる。
- 現在のプレビュー位置を再生終了位置へ設定できる。
- 終了位置の反映では、タイムライン時刻を音源内トリム終了へ正しく変換する。
- 音源長を超える場合は音源末尾へクランプし、開始以前の位置は受け付けない。

### R6: BGM の動画末尾フィット

- BGM 各クリップに `動画末尾に合わせる` 操作を追加する。
- 現在の開始位置から音源が十分長い場合はトリム終了を短くして動画末尾へ合わせる。
- 音源が短い場合は実効長を維持して開始位置を後ろへ移し、動画末尾へ合わせる。
- 変更対象は押したBGMクリップだけとし、他クリップの配置は変更しない。
- 動画長がない場合は操作不可とする。

## 非機能要件・互換性

- `Caption.text` 内の改行を時分割の保存表現とする現行仕様を維持する。
- `bgm` / `bgmClips` の互換ミラー仕様を変更しない。
- ボタンは明示ラベルと `aria-label` / `title` を持ち、モバイルで押しやすい高さを確保する。
- 編集前に既存のプレビュー一時停止処理を通す。
- 外部依存ライブラリは追加しない。

## 受け入れ条件

- 空行時分割で3行を1カードにした後、1行カードへ戻すと3行が3カードとしてプレビューされる。
- 混在入力 `通常A / 時分割A / + 時分割B / 通常B` が3カードになり、中央だけ2行時分割になる。
- 時間指定クリア後も全文章とカード構造が残る。
- 個別設定クリア後、全 override が消え、個別設定バッジも消える。
- BGM／ナレーション終了を現在位置へ設定した時、表示上の再生終了が現在位置と一致する（音源長以内）。
- 動画末尾超過BGMにフィットを実行すると、表示上の終了が動画末尾と一致する。
- 既存の単一行キャプション、`⏎` 形式、BGM保存／復元、プレビュー／書き出しに回帰がない。
