# F4: 長文キャプションの簡単入力 - 仕様書

## 概要

ミュージックビデオの歌詞や長めの字幕のように「多数のキャプションを連続して入れる」作業を、
(1) テキスト一括入力 と (2) 再生しながらのタイミング打ち で簡単にする。
2〜3 個の簡単なキャプションを入れる現在のフローはそのまま維持する。

## 背景・課題

- 現在は「1 個追加 → 時間調整」の繰り返しで、歌詞のような 10〜30 行の入力は非常に手間。
- 入力中に全体像（全行と時間の対応）が見えず、修正もしづらい。
- スマホでの入力が主なので、モバイルで破綻しない UI が必須。PC はレスポンシブで最適化。

## 要件一覧

| # | 要件 | 優先度 | 説明 |
|---|------|--------|------|
| R1 | テキスト一括入力 | 必須 | 複数行テキストを貼り付け/入力 → 1 行 = 1 キャプションとして一括追加 |
| R2 | 自動タイミング割付 | 必須 | 「均等割り」または「1行あたり固定秒」で初期タイミングを自動設定 |
| R3 | タイミング打ちモード | 必須 | 再生しながらボタン 1 つで「今の位置」を区切りとして次々確定（歌詞打ちの定番操作） |
| R4 | 全体像の把握 | 必須 | 一括入力前に行数と割付結果（開始-終了）をプレビュー表示 |
| R5 | モバイル最適化 | 必須 | 大きなタップ領域・縦スクロールのみ・キーボードと干渉しない配置 |
| R6 | Android/PC 限定 | 必須 | standard フレーバーのみ表示 |
| R7 | 既存フロー維持 | 必須 | 1 行入力 +「追加」の既存 UI は変更しない |

## 画面仕様

### (1) 一括入力モーダル（CaptionBulkAddModal）

CaptionSection に「まとめて入力」ボタン（standard のみ）→ フルスクリーン寄りモーダル。

- **テキストエリア**（大きめ・複数行）: 歌詞などを貼り付け。空行は無視。
- **割付方法**（2 択セグメント）:
  - `均等割り`: 現在の再生位置から動画末尾までを行数で等分。
  - `固定（3秒/行）`: 現在の再生位置から 1 行 3 秒で連続配置（末尾は動画長でクリップ）。
- **プレビュー一覧**: 「#, 開始-終了, テキスト」を一覧表示（追加前に確認できる）。
- **追加ボタン**: 一括で captionStore へ追加（既存 addCaption と同じ既定フェード等を適用）。
- 追加後は既存のキャプション一覧で個別編集可能。

### (2) タイミング打ちモード（CaptionSection 内のバー）

キャプションが 2 件以上あるとき、「⏱ タイミング打ち」トグル（standard のみ）でバーを表示。

- バー内容: `対象: n/全N「テキスト先頭…」` + 大きな「⏱ ここで区切る」ボタン + 「終了」。
- 動作: 再生中に「ここで区切る」を押すと、**現在の対象キャプションの endTime と
  次のキャプションの startTime を現在の再生位置に設定**し、対象を次へ進める。
  最後のキャプションでは endTime のみ設定して終了。
- 対象は「選択した行から」開始できる（一覧の行タップで対象変更…v1 は先頭から順でも可）。
- 再生操作自体は既存のプレビューセクションを使用（プレビューが画面上部にあるため、
  スマホでは打鍵バーを画面下部に固定表示 sticky とし、動画を見ながら押せるようにする）。

## データ構造

新規の型・ストア変更なし（既存 `Caption` / `captionStore.addCaption/updateCaption` を使用）。
一括追加用に `captionStore.addCaptions(items: {text, startTime, endTime}[])` を追加（1 アクションで
まとめて追加し、undo 的な再描画コストを抑える）。

## テスト計画

- captionStore.addCaptions のユニットテスト（割付結果がそのまま入ること、空行スキップは UI 側）。
- 割付計算（均等割り/固定秒）の純関数テスト。
- タイミング打ちの状態遷移（end/next start 設定、最終行の扱い）のテスト。
