# F5: キャプション重点ブラッシュアップ - 仕様書

## 概要

F1/F4 で導入したキャプション機能を、実運用（歌詞・長文字幕の大量入力とタイミング調整）に
耐えるレベルへブラッシュアップする。スマホ・PC 両対応（レスポンシブ）を維持し、
**すべて standard フレーバー（Android/PC）限定**とする（iOS Safari は現状 UI を維持）。

## 要件と設計判断

### R1: まとめて入力 — キャプション間の間隔

- 間隔を **0秒 / 300ms / 500ms** の 3 択セグメントで選択（デフォルト 300ms）。
- 均等割り・固定秒の両モードで、連続するキャプションの間に選択した間隔を挿入する。
- 追加後は従来どおり個別に開始/終了を調整可能（間隔は割付時の初期値にすぎない）。

### R2: まとめて入力 — 表示秒数の可変化

- 「3秒固定」ボタンの増殖はせず、**ステッパー（− / 秒数表示 / ＋）+ 直接数値入力**にする。
  - 範囲 0.5〜30 秒、ステップ 0.5 秒、デフォルト 3 秒。
  - スマホでは −/＋ の大きめタップ、PC では数値直接入力が使いやすい。両立する UI。

### R3: まとめて入力 — 再編集ラウンドトリップ

- キャプションが既にある状態で「まとめて入力」を開くと、**現在の全キャプションが
  時間記法付きテキストとしてプリフィルされる**（ボタン表示は「まとめて編集」）。
- テキストを自由に編集（行の追加・削除・文言変更・時間変更）して反映すると、
  キャプションカードへ反映される。
- **反映の対応付けは行順**: i 行目 ↔ 既存 i 番目のキャプション。
  - 既存行は id を保持し、個別スタイル（override）やフェード設定を維持したまま
    テキスト・時間だけ更新する。
  - 追加行は新規キャプションとして作成、行が減った分の既存キャプションは削除される
    （モーダル内に「反映すると現在の n 件を置き換えます」と明示）。

### R4: 行ごとの時間記法（AI 連携フォーマット）

- `@` は歌詞に登場しやすいため不採用。**行頭ブラケット記法**を採用する:

  ```
  [00:03.0-00:07.5] 明日はいい日になるさ
  [01:03-01:12] 恋は@（アットマーク）でも安全
  時間なしの行は自動割付されます
  ```

  - 判定は「行頭が `[時間-時間]` で始まる場合のみ」。歌詞本文が `[` で始まり
    かつ時間形式に一致することは実質なく、本文中の記号は一切影響しない。
  - 時間形式: `MM:SS` / `MM:SS.s` / 秒数のみ（`63.5` 等）。区切りは `-` または `~`。
  - このフォーマットは R3 のプリフィル出力と同一。**外部 AI（音声解析等）に
    「この形式で出力させる」→ 貼り付けで一括登録**、のワークフローを想定する。
- 割付ルール: 時間付き行はその時間をそのまま使用。時間なし行は「直前行の終了 + 間隔」から
  固定秒で配置。均等割りは全行が時間なしの場合のみ適用（混在時は固定秒扱い）。

### R5: UX フロー — 「入力 → タイミング調整」の順番を可視化

- ボタンを **STEP 表記**にし、タイトル下に小さな説明文（約9px、スマホでも1行）を添える:
  - `① まとめて入力` ／ 説明「歌詞や字幕を一括で追加・編集」
  - `② タイミング打ち` ／ 説明「再生しながら切り替えを確定」
- タイミング打ちが押せない理由（キャプション 2 件未満）は disabled 時の説明文で示す。

### R6/R7: タイミング打ち v2 — 途中から・間（ま）対応

- **開始対象は現在のプレビュー位置から自動選択**（再生位置にかかっている、
  または直後のキャプションを対象にする）。毎回 1 件目からやり直す必要はない。
- **◀ / ▶ ボタン**で対象キャプションを前後に移動できる（微修正のやり直し用）。
- 歌詞間の「間」に対応するため **2 フェーズ方式**:
  - フェーズ「終了待ち」: 対象キャプションの終わりで押す。ボタンは 2 つ:
    - `⏭ 区切って次へ` … 終了＝次の開始を同時に確定（間なし・従来動作）
    - `∥ 終了だけ` … 終了のみ確定し、フェーズ「開始待ち」へ（間を空けたいとき）
  - フェーズ「開始待ち」: 次のキャプションが始まる位置で `▶ ここから開始` を押す。
- ガード: 終了は対象の開始より後、開始は対象の終了より前のみ受け付ける（誤爆防止）。
- バーは画面下部固定のまま（動画を見ながら押せる）。

### R8: フォント UI — 固定 3 + ドロップダウン

- 固定ボタン: **ゴシック / 明朝 / 丸ゴシック** の 3 つ。
- それ以外（手書き風・等幅・端末標準、今後の追加分）は **ドロップダウン**で選択。
  ドロップダウン選択中はセレクトボックスがハイライトされる。
- 一括設定・個別設定モーダルの両方に適用。iOS は従来の 2 ボタンのみ。

### R9: サイズのカスタム値

- プリセット（小/中/大/特大 = 56/80/112/148px @1080p 基準）に加え、
  **「カスタム」でスライダー + 数値入力**（24〜240px、1080p 基準）。
- データ: `CaptionSettings.fontSizeCustom?: number | null`（additive・省略時 null）。
- 適用優先度: 個別 override（プリセット）> 一括カスタム値 > 一括プリセット。
- 描画は standard エンジンのみ対応（iOS は無視してプリセットで描画）。

### R10: 位置のカスタム XY

- プリセット（上/中/下）に加え、**「カスタム」で X/Y スライダー**（キャンバスに対する
  % 指定、0〜100%。テキスト中心座標）。デフォルトは中央 X=50 / Y=85 相当。
- データ: `CaptionSettings.positionCustom?: { x: number; y: number } | null`（additive）。
- 適用優先度: 個別 override（プリセット）> 一括カスタム XY > 一括プリセット。
- 描画は standard エンジンのみ対応。

## データ構造（すべて additive）

```ts
// CaptionSettings（types/index.ts, SerializedCaptionSettings にも同フィールド追加）
fontSizeCustom?: number | null;             // px @1080p 基準
positionCustom?: { x: number; y: number } | null;  // % (0-100), テキスト中心

// captionStore
setFontSizeCustom(value: number | null): void
setPositionCustom(value: { x: number; y: number } | null): void
replaceCaptions(items: { id?: string; text; startTime; endTime }[]): void  // R3 用

// captionBulkInput.ts v2
parseBulkCaptionInput(input): BulkCaptionLine[]        // 時間記法パース
formatCaptionsAsBulkText(captions): string             // プリフィル出力
planBulkCaptions(lines, mode, { startTime, totalDuration, fixedDurationSec, gapSec })
```

## テスト計画

- `captionBulkInput`: 時間記法パース（正常系/紛らわしい本文/秒形式）、間隔付き割付、
  整形出力→再パースのラウンドトリップ。
- `captionStyle`: サイズ/位置の解決優先度、範囲クランプ。
- `captionStore.replaceCaptions`: id 保持（override 維持）・追加・削除。
