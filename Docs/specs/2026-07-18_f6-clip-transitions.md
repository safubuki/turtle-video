# F6: クリップ間トランジション + プレビュー音量増幅修正 - 仕様書

standard フレーバー（Android/PC）限定。iOS Safari は非対応（データは無視され通常のカット切替）。

## 1. クリップ間トランジション

### 要件

- 動画・画像カードの**間**にコネクタ（小さな接続ボタン）を表示し、そこからトランジションを設定。
- 種類: **ディゾルブ / フェード(黒) / フェード(白)**、時間: **0.5 / 1 / 2 秒**。

### 改訂 v2: ディゾルブは「重ねる」方式へ（ユーザーフィードバック反映）

静止フレームへの溶け込みでは「動画が止まって見える」ため、ディゾルブは
**本物のオーバーラップ**（前のクリップが流れ続けたまま、次のクリップが d 秒早く始まり
映像・音声ともクロスフェード）に変更した。**そのぶん総再生時間は d 秒短くなる**。

時間変化への対応（不整合防止）は `src/utils/transitionTimeline.ts` を単一ソースとして:

- `computeTransitionTimelineRanges()` / `calculateTotalDurationWithTransitions()` /
  `findActiveTimelineItemWithTransitions()`（オーバーラップ窓では**次のクリップ**を優先）/
  `getTimelineAdvanceForItem()`（cursor 前進量 = duration − overlap）。
- 参照箇所を全て統一: mediaStore の totalDuration（= シークバー最大値・エクスポート尺）、
  standard preview エンジンの区間/アクティブ判定、シークコントローラ、
  visibility 復帰同期、audio session、standard export の音声スケジューリング、
  TurtleVideo の表示区間。**トランジション未使用時は従来の逐次加算と完全一致**
  （回帰テストで保証）。
- オーバーラップは双方のクリップに最低 0.15 秒を残すよう自動クランプ。
- エンジンはオーバーラップ窓で前クリップを「準アクティブ(peer)」として描画（下層）・
  再生継続し、次クリップを上にアルファ合成。音声は双方をクロスフェード。
  エクスポート音声も同じクロスフェードエンベロープで一致させる。
- フェード(黒/白)は従来どおりタイムライン長を変えないディップ方式（選択肢として共存）。
- iOS 縮退: apple-safari エンジンは逐次タイムラインのまま（トランジション無視）。
  ディゾルブ入りプロジェクトを iOS で開くと totalDuration が短いぶん終端が
  切り詰められる（クラッシュはしない）。iOS 展開時に対応する。

### （旧 v1 の設計判断: タイムライン長を変えない — フェードには引き続き適用）

ユーザー指摘のとおり、トランジションで総時間が変わると音声同期・キャプション・
保存データ・エクスポート尺のあらゆる箇所に不整合が波及する。そこで本実装は
**クリップのオーバーラップを行わず、既存のタイムラインの上に「見た目」だけを重ねる**:

- **ディゾルブ**: 現クリップの終端 d 秒間、次クリップのフレームをアルファ 0→1 で重ねる。
  次クリップは Android では境界前にプリロール済み（0-11 preroll）のため、その要素の
  静止フレーム（トリム開始位置付近）を描画する。次クリップ未準備時はフェード(黒)へ縮退。
- **フェード(黒/白)**: 境界前 d/2 秒で色板へディップ、境界後 d/2 秒で明ける。
- ON/OFF しても totalDuration・各クリップの時間帯・音声・キャプションは一切変化しない。
- 音声はトランジションの影響を受けない（必要なら既存のクリップ音声フェードを併用）。

### データ構造（additive）

```ts
export type ClipTransitionType = 'dissolve' | 'fade-black' | 'fade-white';
export interface ClipTransition { type: ClipTransitionType; duration: number; }
// MediaItem.transitionToNext?: ClipTransition | null;  // 「このカード→次のカード」境界の設定
```

- 純ロジックは `src/utils/clipTransitions.ts`（オーバーレイ計算・テスト済み）。
- 描画は standard preview エンジンの renderFrame（プレビュー = エクスポートで同一描画・WYSIWYG）。
- 永続化: `SerializedMediaItem.transitionToNext`（additive・旧データは undefined で従来動作）。
- 最後のカードの transitionToNext は無視（コネクタ自体を表示しない）。
- 複製時は設定ごとコピーされる。

### UI

- カード間の中央に小さなコネクタボタン（未設定:「＋ トランジション」/ 設定済み: 種類と秒数表示）。
- タップでインライン展開: 種類（なし/ディゾルブ/フェード黒/フェード白）+ 時間（0.5/1/2 秒）。
- standard フレーバーのみ表示。ロック中は操作不可。

## 2. プレビュー音量増幅（>100%）が効かないバグ修正

### 原因

standard preview の動画音声は安定性優先で**ネイティブ経路**（`element.volume`）を使うが、
`HTMLMediaElement.volume` は仕様上 **1.0 が上限**のため、100% 超の設定（最大250%）が
プレビューでは無視されていた（エクスポートは WebAudio ゲインなので効く）。
BGM/ナレーションは preview でも WebAudio 経路のため増幅が効く。
→「小さくは効くが大きくは効かない（動画）」というユーザー体感と一致。

### 修正

`getPreviewAudioOutputMode`（standard）に増幅判定を追加:
**基準音量（フェード適用前）が 100% 超の動画は WebAudio 経路へルーティング**し、
ゲインノードで増幅を反映する。100% 以下は従来どおりネイティブ経路（挙動不変）。
フェードで一時的に音量が下がっても経路が切り替わらないよう、判定には
`baseVolume`（フェード前の設定値）を用いる。

## 3. 一括シフトのステッパー改善

- −/＋ ボタンは**長押しで加速**（400ms 後から 100ms 間隔で 1 秒ずつ）。
- 数値の**直接入力欄を併設**（0.1〜600 秒）。−/＋ ボタンは従来どおり残す。
